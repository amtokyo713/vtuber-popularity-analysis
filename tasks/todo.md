# 新人VTuber 人気要因分析プログラム 実装計画

## 1. タスク概要
`https://eve.ebb.jp/yt2/analyze_concurrency.php` の4ページ(page=1〜4)に掲載されている「1年間の同時接続者数ランキング」に載っている新人VTuberを全員分析し、「なぜ人気になったのか」を可能な限り具体的に記載するプログラムを作成する。

## 2. 現在把握しているデータ構造(WebFetchで軽く確認済)

| ページ | 層 | 代表例 | 最高同接の目安 |
|--------|---|-------|--------------|
| page=1 | 最上位 | AZKi, 角巻わため, 博衣こより, 酒寄颯馬(にじさんじ) 等 | 2万〜3万超 |
| page=2 | 中位 | 月宿クド(からくり), 一夜見サヤ, 恋兎るら 等 | 700前後 |
| page=3 | 注目層 | 日ノ隈らん, 花澄れん, 天使サたん 等(約32名) | 300前後 |
| page=4 | 新規参入層 | 夜神ノア, 竜頭だび, 七不思議めいろ 等 | 200前後 |

- 合計で **60〜80名程度** の新人VTuberが対象になる見込み。
- 各エントリに含まれる情報: チャンネル名/最高同接/初配信日/YouTube動画ID(サムネイル用)/(一部)所属事務所タグ。
- **登録者数/チャンネルID/Twitter/TikTokリンクは掲載されていない** ため、別途収集が必要。

## 3. 技術スタックの提案(デフォルト)

| 要素 | 採用案 | 理由 |
|-----|-------|------|
| 言語 | **Python 3.11+** | スクレイピング + 分析 + HTML生成に最適 |
| HTML取得 | `requests` | eve.ebb.jp は静的HTML(SSR)なのでJS実行不要 |
| HTML解析 | `BeautifulSoup4` + `lxml` | 安定して高速 |
| YouTube情報 | `yt-dlp`(APIキー不要)でチャンネル名・登録者数・説明欄を取得 | Data API v3 を使わず無料で完結 |
| Web調査 | **Claude CodeのWebSearch/WebFetchツール**を呼び出す方式で実装 | 追加APIキー不要、検索結果から直接分析 |
| レポート | Jinja2 + Vanilla CSS で単一HTMLファイル出力 | プレビュー可能、配布容易 |

## 5. 実装ステップ (チェック可能な形式)

### Phase 0: 基盤準備
- [ ] `requirements.txt` を作成 (requests, beautifulsoup4, lxml, yt-dlp, jinja2)
- [ ] `.claude/launch.json` を作成 (Pythonの簡易HTTPサーバーでプレビュー可能にする)

### Phase 1: スクレイピング (`src/scraper.py`)
- [ ] 4ページのHTMLを取得してローカルキャッシュ(`data/cache/`)
- [ ] 各カードから以下を抽出:
  - チャンネル名
  - 最高同接数(数値)
  - 初配信日時
  - YouTube動画ID
  - 所属事務所タグ(存在する場合)
- [ ] 重複エントリを排除(同一チャンネルの別動画があるため)
- [ ] `data/channels_raw.json` に保存

### Phase 2: YouTube情報取得 (`src/youtube_fetcher.py`)
- [ ] 各動画IDから yt-dlp で以下を取得:
  - チャンネル名・チャンネルID
  - チャンネル登録者数
  - 動画タイトル・投稿日
  - 動画の説明欄(Twitter/TikTokリンク抽出用)
- [ ] 説明欄から正規表現で Twitter(X), TikTok, Twitch 等のリンクを抽出
- [ ] `data/channels_enriched.json` に保存

### Phase 3: Web調査 (`src/researcher.py`)
各VTuberについて以下のクエリでWeb検索を実行し、結果を収集:
- [ ] `"{チャンネル名}" VTuber`
- [ ] `"{チャンネル名}" 初配信 感想`
- [ ] `"{チャンネル名}" 切り抜き`
- [ ] `"{チャンネル名}" まとめ`
- [ ] `"{チャンネル名}" 前世` or `中の人`
- [ ] `"{チャンネル名}" バズ`
- [ ] `"{チャンネル名}" TikTok`
- [ ] `"{チャンネル名}" Twitter` or `X`
- [ ] 検索結果のURLから上位3件をWebFetchで内容取得・要約
- [ ] `data/channels_analyzed.json` に追加保存

### Phase 4: 成功要因分類 (`src/analyzer.py`)
収集情報から以下のパターンを自動判定:
- [ ] **事務所パワー型** (ホロライブ/にじさんじ/ぶいすぽ等の大手事務所初配信特需)
- [ ] **前世強豪型** (前配信者/歌い手/声優出身で既存ファンが流入)
- [ ] **企画バズ型** (初配信企画・コラボ・歌ってみた等が話題)
- [ ] **ビジュアルバズ型** (デザイン・イラストレーターの話題性)
- [ ] **切り抜き/TikTok拡散型** (切り抜きチャンネル多数・TikTok動画が拡散)
- [ ] **Twitter(X)バズ型** (特定ツイートの拡散が流入に直結)
- [ ] **SNS×企画複合型** (複数要因の組合せ)

### Phase 5: HTMLレポート生成 (`src/reporter.py`)
- [ ] 全体統計セクション(成功パターン別件数・事務所別分布)
- [ ] VTuber個別カード(サムネ/チャンネル名/同接/分類/詳細分析テキスト)
- [ ] 同接順・事務所別・成功パターン別のフィルタリングUI
- [ ] `output/report.html` を生成
- [ ] `output/preview.html` にコピー(プレビュー用)

### Phase 6: 実行と検証
- [ ] `python src/main.py` を実行して全Phase通す
- [ ] `preview_start` で launch.json のサーバーを起動
- [ ] `preview_screenshot` で見た目を確認
- [ ] ユーザーに結果を報告

## 6. 【重要】制約事項と率直な見積もり

### ⚠️ できること
- ✅ 4ページのスクレイピングで全VTuberのリスト化
- ✅ yt-dlpで各チャンネルの登録者数・Twitter/TikTokリンク取得
- ✅ Web検索で各VTuberの公開情報(まとめ記事・切り抜き・ニュース)を収集
- ✅ 成功パターンの分類

### ⚠️ できないこと・制限
- ❌ **TikTokの具体的バズ動画URL**までのピンポイント特定は、TikTok API非利用のため困難。「まとめサイトで言及されたTikTok話題」を間接的に記載する形になる
- ❌ **Twitterの具体的バズツイートID**も、X API有料化のため直接取得不可。「切り抜き/まとめで触れられた話題」ベースの記載になる
- ❌ 「なぜ人気になったか」の断定は不可能(公開情報からの推測に留まる)
- ⚠️ 60〜80名全員についてWebSearchを回すと、検索回数が400〜800回程度必要で時間がかかる

### 現実的対応案
- **上位層(page=1)の20名は詳細分析**(各8クエリ)
- **中位層(page=2)は中程度分析**(各4クエリ)
- **注目/新規層(page=3,4)は基本分析**(各2クエリ)
